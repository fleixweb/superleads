# Claude Code 宿主网页可访问性基准

- 日期：2026-07-19
- 宿主：Claude Code（Linux WSL2 会话）
- 方向：英国「不锈钢保温杯 / 可重复使用水瓶」经销商 / 批发商发现（仅用于测量宿主与来源可访问性）
- 样本量：8 个固定公开 URL + 3 条工具路径

## 本文是什么 / 不是什么

- 这是**宿主工具与来源可访问性**的测量，用于校准默认发现在真实网络下的执行摩擦。
- **不是**客户质量、采购意向或商业价值评估。
- **不是**全网覆盖率或“英国有多少家保温杯经销商”的结论。
- 403 / 404 / 415 / 429 / JS 空壳只表示“本会话此工具未取得正文”，**不表示企业不存在、不相关或不值得开发**。
- 未使用 example.com 等占位数据冒充真实客户；本表所有条目均为真实请求结果。
- 未绕过登录 / 验证码 / 付费墙，未使用私有接口，未安装任何 MCP 或新依赖。未记录 cookie、token、请求密钥或网页正文快照。

## 测量的工具路径

| 工具路径 | 类型 | 本会话状态 |
|---|---|---|
| WebSearch（原生） | 搜索结果 | **全程 429 限流**：本会话 0 次成功搜索 |
| WebFetch（原生） | 单页只读打开（自动升级 HTTPS，跨主机重定向不跟随） | **全程 429 限流**；此前另见 403 / 404：本会话 0 次成功取得业务正文 |
| curl（Bash 沙箱内已安装，只读 GET） | 单页只读打开，跟随同源重定向 | 沙箱有网络，可取得正文；结果见下表 |

> 关键差异：本会话原生 WebSearch / WebFetch 因 Anthropic 侧速率限制不可用，而沙箱内 `curl` 的只读 GET 可用。二者是**独立的宿主能力**。任一工具路径只有在实际返回安全公开 URL、只读 GET（或等效可追溯读取）记录、非空可用正文摘录及定位信息，并满足当前 Source/Observation 契约时，才能形成 `source.open` Observation；否则只能记为发现线索、访问失败或来源受限，不能冒充 Observation。

## 汇总

- **业务正文可得率（curl）**：4/8 = 50%（含 1 个控制样本）；真实目标站 3/7 ≈ 43%。
- **公开联系方式可提取**：3/8 拿到电话或邮箱；其中带公开邮箱 2/8（chillys、kleankanteen）。
- **失败/受限分布**：
  - 429（原生工具限流）：WebSearch、WebFetch 本会话全部请求。
  - 403（反爬拒绝）：yell.com。
  - 415（媒体类型拒绝，等价拒爬）：onegreenbottle.com。
  - 202 + JS 空壳（无服务端正文）：europages.co.uk。
  - 200 但登录墙/JS 无业务正文：linkedin.com。
  - 404（历史测试中的错误路径）：不在本固定集内。

## 对默认发现的含义

1. 真实一轮里，相当比例的来源会 403 / 415 / JS 空壳 / 登录墙。默认发现必须把这些如实记为 `source_restricted` / `not_observed` 并保留 Candidate，而不是删除或判定“无此客户”。
2. 在本次小样本下，品牌官网（Shopify 类）比聚合目录（europages / yell / linkedin）更易取得正文与公开联系方式；目录站在本样本中常见反爬或 JS 渲染。这只是当前执行策略提示：默认发现可优先尝试官网/联系页，并把目录站作为线索发现路径，不是普遍商业价值或客户质量结论。
3. 线索产量天花板由**宿主可用的取页路径**决定，不由 Superleads 协议决定。当原生 WebFetch 受限时，沙箱 `curl` 只读 GET 是可用的替代路径；两者皆不可用时应降级为研究计划或纯发现候选池。

## 复现方法

固定 URL 集与逐条 HTTP 状态、正文字节、业务关键词命中数、公开邮箱计数、联系入口命中为上表。测量仅用只读 GET 与标准 UA，无 cookie / token / 登录态，不保存网页正文。原生 WebSearch / WebFetch 的 429 状态为 Anthropic 侧速率限制，可能随时间恢复；建议后续在非限流时段复测原生工具路径以补齐对比。
