# Product Profile Input

Collect only what helps the current task. Do not force a factory/OEM/industry questionnaire.

Useful fields:

- Product or service name.
- Target country or region.
- Target customer type or channel.
- Application context or downstream use.
- Keywords and excluded words.
- Seed companies, seed URLs, competitors, brands, or reference sites.
- Desired contact detail level.
- Target lead count and output mode.
- Must-verify claims.
- Existing table or source material paths.

## Current development boundary

Capture these only in the current Brief as free text; none is a fixed product,
industry, role, or channel enum:

- What the user is offering and any supply boundary.
- Commercial positioning or packaging/authorization context.
- Intended use, application, or business boundary.
- What counterparties should do in this task.
- What should not enter this task's customer pool.
- How named competitors, brands, manufacturers, or reference companies should
  be handled.
- The public signal needed to treat an Entity as matching, and the public
  signal that would make it out of scope.

Only retain unanswered questions that can reverse the customer direction. If
they remain unresolved, use a provisional direction and show samples only.
